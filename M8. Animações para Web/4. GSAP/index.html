    Sobre o GSAP¶

    GSAP (GreenSock Animation Platform) é considerado o padrão da indústria para animações web profissionais:

    • ⚡ Performance superior - 20x mais rápido que CSS animations
    • 🎯 Controlo preciso - Frame-perfect animations
    • 🔧 ScrollTrigger poderoso - Animações baseadas em scroll avançadas
    • 📱 Cross-browser - Funciona em todos os browsers
    • 🎨 Flexibilidade total - Anima qualquer propriedade CSS

    O GSAP é uma ferramenta de código pensada para criar animações para a web, desde as mais simples e lineares até às mais complexas e com dependências. Com recurso a plugins consegue diversos efeitos de animação e diferentes tipos de triggers, como por exemplo os baseados no scroll da página.

    Showcase¶

    Aqui estão alguns exemplos de sites com secções animadas desenvolvidas com recurso ao GSAP, https://good-fella.com/, https://www.felix-nieto.com/ e https://www.museumofmoney.com/

    Instalação¶

    Para testar o GSAP podemos criar um codepen, adicionar o package GSAP e fica logo disponível para ser usado. Podes começar com este GSAP Starter

    Sendo uma framework agnóstica pode ser instalada de diversas formas conforme a tecnologia web usada. No nosso caso vamos usar a versão para desenvolvimento web nativo e por isso basta indicar apenas o script disponibilizado para o efeito. Deverá ser colocado antes dos nossos scripts que vão usar o GSAP

    <html>
  <head>
     ...
  </head>

  <body>
    <div className="animateMe"></div>
    <h1>Hello GSAP!</h1>

    <!-- Put your script after the HTML -->
    <script src="https://cdn.jsdelivr.net/npm/gsap@3.14.2/dist/gsap.min.js"></script>
    <!-- And put your own code after - any GSAP code you write needs to have access to the library -->
    <script>
    // custom code using GSAP
    </script>
  </body>
</html>

    Se necessitarmos de usar plugins será necessário adicionar outros links tal como indicado aqui

    Easing¶

    O easing de uma animação é determinante para uma correta execução da mesma. O easing determinada forma como as propriedades são a alteradas nos elementos no durante o tempo da animação. Para além o easing linear e do ease in/out o GSAP tem diversas possibilidades para a curva do easing, sendo a curva power1.in o valor por defeito. Nesta página podemos visualizar os diversos tipos de easing.

    A escolha acertada do easing para uma determinada animação pode ser suficiente para que o resultado final seja o desejado.

    De seguida são apresentados alguns tipos de easing menos habituais com exemplos práticos de utilização

    Controlo¶

    Uma animação GSAP pode ser criada e controlada em termos da sua execução. Podemos colocar a animação numa variável e depois executar play(), stop(), pause() e reverse(). Por exemplo na animação do menu lateral podemos criar uma versão melhorada usando estes controlos. Nota sobre a propriedade pausedque garante que a animação não é imediatamente executada.

    let tween = gsap.to("nav", { duration: 0.5, ease: "back", x: -30, paused: true })

const openMenu = () => tween.play()
const closeMenu = () => tween.reverse()

    Timelines¶

    Uma timeline em GSAP é uma sequência de animações (tweens) que por defeito são executados uma após a outra.

    Para exemplificar podemos melhorar o exemplo do menu lateral ,onde para além da animação do menu e dos items, podemos incluir também animação para os ícones de abertura e de fecho do menu, com scale.

    let anim = gsap.timeline({paused: true})
  .to("nav", { duration: 0.5, ease: "back", x: -30 })
  .from("nav p", {xPercent: -100, stagger: 0.1, ease: "back"})
  .from("nav button", { scale: 0, ease: "back"})


const openMenu = () => anim.play()
const closeMenu = () => anim.reverse()

    Neste exemplo, cada tween da timeline é executado um a seguir ao outro. Se pretendermos algum desfasamento ou sobreposição de tweens, a timeline disponibiliza um parâmetro de posição para o efeito.

    Na animação acima podemos trabalhar melhor o arranque das animações e criar alguma sobreposição no arranque das mesma, como por exemplo: 

    let anim = gsap.timeline({paused: true})
  .to("nav", { duration: 0.5, ease: "back", x: -30 })
  // executa a animação dos <p> 0.25seg antes da timeline terminar
  .from("nav p", {xPercent: -100, stagger: 0.1, ease: "back"}, "-=0.25")
  .from("nav button", { scale: 0, ease: "back"})


const openMenu = () => anim.play()
const closeMenu = () => anim.reverse()

    Existem diversos parâmetros de posição, mas os mais frequentes são: “-=1” - inicia 1 seg antes da timeline terminar
    “+=1” - inicia 1 seg depois da timeline terminar
    “<” - arranca quando o tween anterior iniciar
    “<1” - inicia 1seg depois do tween anterior iniciar
    “1” - inicia exatamente ao segundo 1 da timeline

    Os valores indicados estão em segundos, mas nas novas versões do GSAP também permitido a utilização de %. Por exemplo, "<25%", significa que o tween vai iniciar após o início do tween anterior em 25% da sua duração.

    [!NOTE] GSDevTools Para realizar uma análise mais detalhada da animação, os momentos em que cada tween entra e termina, efetuar debug de algo que está desenquadrado podemos usar esta ferramenta que disponibiliza uma playhead da animação, com diversas funcionalidades. Deverá ser efetuada a sua instalação e adicionada a seguinte linha de código para uma determinada animação GSDevTools.create( {animation: anim});.

    Aqui podes consultar um exemplo prático de animação de uma secção hero de um portfolio.

    let anim = gsap.timeline()
  .to(".hero", {clipPath: "polygon(0% 0, 100% 0, 100% 100%, 0% 100%)", duration: 0.75 })
  .fromTo(".triangle", {xPercent: -100}, {xPercent: -10, duration: 1})
  .to("p.m1", {text: "Olá!"})
  .to("p.m2", {text: "Design for life."})
  .from(".pen", {y: 400})
  .to(".triangle", { x: +50}, "+=1")
  .to(".pen", { x: +50}, "<")

    Quando queremos aplicar a mesma animação a diversos elementos mas individualmente é normalmente criado uma timeline para cada um deles. No exemplo abaixo voltamos ao exemplo do menu lateral, mas desta vez temos uma animação de hover sobre os items do menu. Este poderia ser um caso onde a utilização de transitionpoderia ser suficiente, mas a título ilustrativo vamos neste caso usar uma animação GSAP.

    Em vez de usar o :hoverem css vamos usar eventos javascript para determinar o evento do rato ao entrar num elemento e ao sair de um elemento.

    Usamos um loop para criar os eventos e as animações para cada um dos items do menu

    let items = document.querySelectorAll("nav p")
items.forEach((item) => {
  item.onmouseenter = () => gsap.timeline().to(item, {scaleX: 1.5, duration:0.2})
  item.onmouseleave = () => gsap.timeline().to(item, {scaleX: 1, duration:0.2})
})

    ScrollTrigger¶

    A propriedade scrollTriggerpermite um controlo das animações com base no scroll da página. Encontra-se disponível através de um plugin que deverá ser instalado previamente.

    Na sua versão mais simples podemos aplicar a propriedade scrollTriggera um elemento com neste exemplo:

    gsap.registerPlugin(ScrollTrigger)  

gsap.from(".card" , {  
    xPercent: "-100",  
    scrollTrigger: ".card"  
})

// OR 

gsap.from(".card" , {
  x: "-50vw",
  scrollTrigger: {
    trigger: ".card",
    markers: true,
  }
})

    A animação neste caso vai iniciar durante o scroll quando o topdo elemento trigger(.card) passar o bottomdo viewport.

    [!NOTE] Markers Activar a propriedade markers é uma ótima forma de efectuar debug das aimações no que respeita ao seu arranque e término durante o scroll e o scroll back

    start e end¶

    As propriedades starte end são propriedades do scrollTrigger que determinam o arranque e o término do animação. Por exemplo:

    gsap.from(".card" , {
  x: "-50vw",
  scrollTrigger: {
    trigger: ".card",
    markers: true,
    start: "top 75%",
    end:"bottom 25%
  }
})

    Neste caso a animação inicia quando o top do elemento a animar passa 75% do viewport e termina quando o bottom do elemento a animar passa 25% do viewport

    toggleActions¶

    São as ações que podemos definir, relativamente à animação, para cada um dos eventos implícitos nesta propriedade.

    `toogleActions: "restart pause reverse reset"

    Os eventos implícitos para estas ações estão por ordem: onEnter, onLeave, onEnterBack e onLeaverBack.

    As ações mais utilizadas são: play
    pause
    resume
    reset
    restart
    complete
    reverse
    none

    scrub & pin¶

    Até agora o scroll da página desencadeia uma animação conforme o estabelecido nas propriedades de starte end. Ou seja, durante o scroll o elemento trigger faz desencadear a animação mas não a controla. Com estas propriedades o progresso do scroll fica ligado ao progresso da animação.

    scrub - liga o scroll com o progresso da animação
    pin - fixa o scroll do container da animação durante a sua execução.

    As animações podem ficar complexas e ou até com algum ruído visual com estas propriedades se não forem aplicadas com moderação.

    gsap.registerPlugin(ScrollTrigger) 

gsap.from(".card" , {
  x: "-60vw",
  scrollTrigger: {
    trigger: ".card_container",
    markers: true,
    start: "top 0%",
    end: "bottom 50%",
    scrub: true,
    pin: true,
    //pinSpacing:false
  }
})

    Exemplo Reveal word by word¶

    Split das palavras do text num array para ser aplicado ao GSAP através da class char.

    <style>
      body {
        font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif;
      }

      .text-section {
        min-height: 100vh;
        display: flex;
        flex-direction: column;
        justify-content: center;
        align-items: center;
        padding: 50px;
        text-align: center;
        background: #f8f9fa;
      }

      .split-text {
        font-size: 3rem;
        font-weight: bold;
        margin-bottom: 2rem;
        color: #333;
      }

      .char {
        display: inline-block;
        opacity: 0;
        transform: translateY(100px);
      }
    </style>
  </head>

  <section class="text-section">
    <h2 class="split-text">Um relógio para a sua saúde</h2>
  </section>

  <script>
    gsap.registerPlugin(ScrollTrigger);

    // 1. Split text em caracteres
    function splitTextIntoChars(selector) {
      const element = document.querySelector(selector);
      const text = element.textContent;
      element.innerHTML = "";

      text.split(" ").forEach((char) => {
        const span = document.createElement("span");
        span.textContent = char + "\u00A0";
        span.className = "char";
        element.appendChild(span);
      });
    }

    // 2. Aplicar split
    splitTextIntoChars(".split-text");

    // 3. Animar caracteres
    gsap.to(".char", {
      y: 0,
      opacity: 1,
      duration: 0.8,
      stagger: 0.2,
      ease: "power2.out",
      scrollTrigger: {
        trigger: ".split-text",
        start: "top 80%",
      },
    });
  </script>
</html>

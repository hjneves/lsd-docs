    4. Interfaces modernas

    1. Introduction & Context (30 min)¶

    Goal: Set the stage, explain why CSS frameworks matter, and introduce Bootstrap.

    • Why CSS Frameworks?
      • Speed up development, ensure consistency, responsive design, community support.
      • Show examples of popular sites using Bootstrap (e.g., Twitter, Spotify, Udemy).

    1. Introduction & Example Overview (10 min)¶

    Goal: Present the final product and break it down into manageable parts.

    • Show the Final Design:
      • Display a mockup of the smartwatch landing page (see example layout below).
      • Highlight key sections: navbar, hero, features, testimonials, footer.
    • Discuss Responsiveness:
      • Show how the layout changes on mobile, tablet, and desktop.
    • Explain the Process:
      • We’ll build this step-by-step, focusing on Bootstrap’s grid and components.

    2. Setup & Starter Files (5 min)¶

    Goal: Ensure everyone has the necessary files and is ready to code.

    • Provide Starter Files:
      • Basic HTML template with Bootstrap CDN included.
      • Folder with images (smartwatch product images, logos).
    • Quick Check:
      • Ask students to open the template in their editor and browser.

    3. Step-by-Step Build (60 min)¶

    A. Navbar (10 min)¶

    Goal: Create a responsive navbar with logo, links, and a call-to-action button.

    • Code Along:
      • Use Bootstrap’s navbar component.
      • Add logo, nav links, and a “Buy Now” button.
      • Make it collapse on mobile.
    • Key Points:
      • navbar-expand-lg, navbar-dark, bg-dark
      • navbar-toggler for mobile

    B. Hero Section (15 min)¶

    Goal: Build a full-width hero with a product image, headline, and CTA.

    • Code Along:
      • Use a container and row/col for layout.
      • Add a jumbotron or custom hero with background image.
      • Include a headline, subhead, and “Learn More” button.
    • Key Points:
      • py-5, text-center, btn-primary, btn-lg
      • Responsive image with img-fluid

    C. Features Section (15 min)¶

    Goal: Create a 3-column features section using Bootstrap cards.

    • Code Along:
      • Use a container and row with 3 col-md-4 columns.
      • Each column contains a Bootstrap card with icon, title, and description.
    • Key Points:
      • card, card-body, text-center
      • mb-4 for margin-bottom

    D. Testimonials (10 min)¶

    Goal: Add a testimonial carousel.

    • Code Along:
      • Use Bootstrap’s carousel component.
      • Add 2-3 testimonials with user images and quotes.
    • Key Points:
      • carousel, carousel-item, d-none d-md-block for responsive images

    E. Footer (10 min)¶

    Goal: Build a simple footer with links and copyright.

    • Code Along:
      • Use bg-dark, text-white, and py-4.
      • Add columns for “Product”, “Company”, “Support”.
    • Key Points:
      • row, col-md-3, text-muted

    4. Customization & Responsiveness (10 min)¶

    Goal: Teach students how to override Bootstrap and ensure the page looks good on all devices.

    • Custom CSS:
      • Add a <style> section to override colors, fonts, or spacing.
    • Responsive Checks:
      • Use Chrome DevTools to test on different screen sizes.
      • Discuss common issues and fixes.

    5. Group Activity & Presentation (5 min)¶

    Goal: Students share their work and get feedback.

    • Pair/Group Up:
      • Students compare their pages with a neighbor.
    • Volunteers Present:
      • 1-2 students share their screen and explain their approach.

    Starter Code Snippet¶

    Copy

    <!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>SmartWatch Landing Page</title> <!-- Bootstrap CSS --> <link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.0/dist/css/bootstrap.min.css" rel="stylesheet"> <!-- Custom CSS --> <style> .hero { background: url('smartwatch-bg.jpg') no-repeat center center; background-size: cover; } </style> </head> <body> <!-- Your code here --> <!-- Bootstrap JS Bundle with Popper --> <script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.0/dist/js/bootstrap.bundle.min.js"></script> </body> </html>

    Tips for Success¶

    • Encourage Experimentation: Let students try different Bootstrap classes.
    • Debug Together: When something breaks, use it as a teaching moment.
    • Provide Cheat Sheets: Print or share a Bootstrap classes cheat sheet.

    Updated: Customization with CSS Variables (10 min)¶

    Goal:¶

    Teach students how to customize Bootstrap’s default styles using CSS variables, ensuring their design is consistent and easy to maintain.

    1. Introduce CSS Variables in Bootstrap (2 min)¶

    • Explain: Bootstrap 5 uses CSS variables for colors, breakpoints, spacing, and more. These are defined in the :root selector.
    • Show: Open the browser’s dev tools and inspect Bootstrap’s :root variables (e.g., --bs-primary, --bs-body-color).

    2. Override Bootstrap Variables (5 min)¶

    How to do it:

    • Option 1: Override variables before Bootstrap’s CSS file:

      Copy

      <style> :root { --bs-primary: #fd7e14; /* Change primary color */ --bs-body-bg: #f8f9fa; /* Change background */ --bs-font-sans-serif: 'Roboto', sans-serif; /* Change font */ } </style> <!-- Bootstrap CSS --> <link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.0/dist/css/bootstrap.min.css" rel="stylesheet">

    • Option 2: Use a custom SCSS file (if using SASS):

      Copy

      $primary: #fd7e14; @import "bootstrap/scss/bootstrap";

    Key Variables to Customize:

    Variable

    Purpose

    --bs-primary

    Primary color (buttons, links)

    --bs-secondary

    Secondary color

    --bs-body-bg

    Background color

    --bs-body-color

    Default text color

    --bs-font-sans-serif

    Font family

    --bs-border-radius

    Border radius

    3. Practical Example (3 min)¶

    Task: Change the theme colors and font for the smartwatch landing page.

    • Set primary color to #fd7e14 (orange).
    • Set body background to #f8f9fa (light gray).
    • Change the font to 'Roboto', sans-serif.

    Code Along:

    Copy

    <style> :root { --bs-primary: #fd7e14; --bs-body-bg: #f8f9fa; --bs-body-color: #212529; --bs-font-sans-serif: 'Roboto', sans-serif; --bs-border-radius: 0.5rem; } body { font-family: var(--bs-font-sans-serif); } </style>

    4. Live Preview & Discussion (5 min)¶

    • Show the changes in the browser.
    • Discuss:
      • Why use CSS variables? (Consistency, easy updates, less code)
      • How to find all available variables? (Bootstrap docs)
    • Encourage students to experiment with other variables (e.g., --bs-success, --bs-border-color).

    5. Bonus: Dark Mode Toggle (Optional, 5 min)¶

    For advanced students:

    • Add a button to toggle dark mode using CSS variables:

      Copy

      <button onclick="document.documentElement.setAttribute('data-bs-theme', 'dark')">Dark Mode</button> <button onclick="document.documentElement.setAttribute('data-bs-theme', 'light')">Light Mode</button>

    • Note: Requires Bootstrap 5.3+ and the data-bs-theme attribute.

    Why This Matters¶

    • Efficiency: Change colors, fonts, and spacing in one place.
    • Maintainability: Easier to update and manage.
    • Best Practice: Recommended by Bootstrap for customization.

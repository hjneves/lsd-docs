No editor de blocos interno (Gutenberg) existem alguns conceitos importantes a compreender para uma correcta utilização o mesmo.

# Blocos reutilizáveis

O editor de bloco como o próprio nome indica tem na sua função um conjunto de blocos que são reutilizáveis em diversas páginas. São por exemplo, botões, cartões, parágrafos, etc. Uma vez definidos podem ser utilizados em diversas páginas, com a vantagem de todas as alterações serem propagadas em todos os locais onde forma utilizados.

O Wordpress já traz por defeito um conjunto alargado de blocos para satisfazer a maior parte das necessidades de webdev.

![[Pasted image 20260112153818.png|280]]

Existem diferentes categorias de blocos:
- **Text** - onde os parágrafos e os heading serão os mais usados
- **Media** - para incluir imagens e video
- **Design** - botões e blocos para organização do layout, com uma relação direta com a Flexbox

## Patterns

Os *patterns* são componentes — conjunto de blocos reutilizáveis — organizados para uma determinada função, por ex. um formulário, uma secção Hero, etc. São acelerados de conteúdos para as páginas.
Um detalhe importante nos *patterns* é a capacidade de poderem ou não estar sincronizados. Se um *pattern* estiver com a opção sync ativa vai refletir as suas alterações em todas as páginas onde está a ser utilizado.

Quando um *pattern* está sincronizado (synced), para o conteúdo poder ser diferente em cada utilização por cada página, tem de ser ativada a opção `Overrides`na secção avançada do bloco em causa. Tipicamente utilizado em blocos de conteúdo, com parágrafos ou imagens.

Os *patterns* estão organizados por categorias e dependem também dos temas instalados.

## Templates
Os templates estão relacionado com a estrutura do site. Todas as páginas e posts no wordpress estão assentes num template e por norma as páginas têm por base o template `Pages`

A necessidade de criar um template advém do tipo de páginas que o nosso site tem. Se o site tiver sempre a mesma estrutra à partida bastará um template.

Na estrutura de um template a secção `Content`é a secção onde será apresentado o conteúdo da página, ou seja, o conteúdo que definimos na criação das páginas através de componentes e *patterns*.

![[Pasted image 20260112171915.png]]

> [!NOTE] Nota
> Todos os componentes de conteúdo que forem adicionados a um template, serão apresentados nas páginas que têm este template associado.


| **Templates**                       | **Patterns**                         |
| ----------------------------------- | ------------------------------------ |
| Define a estrutura de uma página    | Secções/componentes dentro da página |
| Aplicado  automaticamente à página  | Inserido manualmente onde precisas   |
| Define o layout geral               | Define o um determinado conteúdo     |
| Ex: "Todas as páginas de trabalhos" | Ex: "formulário de contacto"         |



# Estratégia criação de UI

Uma estratégia para a criação dos layouts de UI no WordPress será o seguintes:
1. Escolher um Tema minimalista e não um tema pré-definido ( a não ser que tenham encontrado um Tema que faça match como que pretendem) . Ambos deverão ser baseados em editor de blocos.
2. Analisar os layouts e identifiar os components criados que podem ser reutilizáveis: header, footer, CTA, cartões, formulários, ...)
3. Criar/Editar os componentes e patterns de acordo com o layout
4. Criar as páginas utilizando os compontes
